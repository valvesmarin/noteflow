# Gestión del Proyecto - NoteFlow

## Organización del Proyecto

Para gestionar el desarrollo de NoteFlow de forma profesional y organizada, utilicé la metodología **Kanban** a través de un tablero en Trello.

### Estructura del tablero:

- **Backlog**: Ideas y funcionalidades futuras
- **Todo**: Tareas pendientes por empezar
- **In Progress**: Tareas en las que estoy trabajando actualmente
- **Review**: Tareas terminadas esperando revisión
- **Done**: Tareas completadas

### Funcionalidades principales divididas en subtareas:

**MVP (Versión 1.0)**
- Configuración inicial del proyecto con Expo + TypeScript
- Configuración de Expo Router con navegación por pestañas (Tabs)
- Implementación del estado global con Zustand + persistencia con AsyncStorage
- Pantalla de Notas (crear, listar y eliminar)
- Pantalla de Checklists (crear, añadir ítems y marcar como completados)
- Pantalla de Ideas (crear, etiquetas y colores)
- Diseño moderno con React Native Paper
- Documentación técnica completa

**Funcionalidades futuras (Backlog)**
- Búsqueda global en todas las pantallas
- Animaciones avanzadas con Reanimated
- Notificaciones push
- Sincronización en la nube
- Sistema de archivado de notas

## Metodología elegida

Elegí **Kanban** porque permite visualizar el flujo de trabajo de forma continua y flexible, sin la rigidez de los sprints del Scrum. Al tratarse de un proyecto individual y académico, Kanban se adapta mejor a mi ritmo de desarrollo y me permite mover las tareas según el progreso real.
